This package contains the code to mine the game of life with multiple threads.
Design options:
  1. Single Responsibility components with job queue.
    To reduce the system requirements the job queue will be entirely in memory.
    Threads:
    • ThreadManager
      • GameRunner [miner]
    • ThreadManager
      • ResultFilter
    The Managers are chained together to create a pipeline without the need for saving jobs into a database.
    The downside to this design is that running the game is quick. Filtering out all the results we don't want, duplicates, non looping states, etc takes the most amount of time.
  2. Multi pipeline threads.
    Each thread is responsible for delivering one piece of input from start to finish.
